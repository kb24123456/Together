# PRODUCT_SPEC.md

## 1. 文档职责
- 本文件是项目「一起 / Together」的唯一产品规格主文档。
- 负责统一：产品定位、MVP 范围、信息架构、页面需求、版本路线。
- 不负责：工程实现细则、Codex 执行规则、视觉与动效设计细则。

## 2. 一句话定位
- 一款以单人工作与日常待办管理为核心、后续可扩展到双人协作的 iPhone Todo 效率工具。

## 3. 当前产品结论

### 事实
- 旧逻辑把产品定义为双人生活协作工具，单人更像试用态。
- 当前主轴已切换为单人 Todo 效率工具。

### 结论
- V1 优先做真实可用的单人 Todo 主链路。
- V2 作为差异化扩展引入双人协作，但仍以 Todo 为核心。
- V3 多人 Space 仅做底层预留，不作为当前承诺能力。
- 决策、纪念日、邀请、绑定、关系页降级为后续附加模块。

## 4. 目标用户
- 事务密度较高、需要持续处理工作待办和项目推进的个人用户。
- 对信息层级、交互流畅度、原生体验有较高要求的 iPhone 用户。

## 5. 核心场景
- 每天进入 Today 首页查看今天最重要的事项。
- 快速收集临时任务，稍后整理进清单或项目。
- 在项目维度查看推进状态。
- 在周视图 / 月视图中查看时间分布与截止压力。
- 在移动端高频完成、重排、筛选、重新聚焦。

## 6. 核心价值主张
- 不只是记录待办，而是帮助用户更快进入“今天先做什么”的执行状态。
- 单人模式本身成立，即使不使用双人模式也具备独立价值。
- 未来可平滑升级为双人协作，而不需要重做产品主逻辑。
- 通过克制、丝滑、原生的 iOS 交互体验形成差异化。

## 7. 当前 MVP 范围

### 一级结构
- Today
- 清单
- 项目
- 日历
- 我

### 全局入口
- 快速创建任务
- 搜索
- 筛选 / 排序 / 分组
- 任务详情

### 核心能力
- Task：创建、编辑、完成、重新打开、提醒、筛选、排序。
- List：系统清单与自定义清单。
- Project：项目状态切换、项目详情、项目内任务聚合。
- Calendar：周视图、月视图、按日期查看任务。
- Profile：通知、主题、默认首页、未来双人模式入口占位。

### 当前明确不做
- 不把双人绑定流做成首版主链路。
- 不把共同决策、纪念日做成首页主功能。
- 不做多人工作组能力。
- 不做社区、内容流、开放社交。
- 不做只为了视觉好看却不提升效率的复杂交互。

## 8. 信息架构原则
- 当前信息架构以单人 Todo 为主，不做双人优先展开。
- Task 是 IA 核心对象；清单、项目、日历是不同观察维度。
- 产品层当前不主动教育用户 Space 概念。
- V1 用户只感知“我的任务空间”。
- 技术层允许后续演进为 `SingleSpace / PairSpace / MultiSpace`。

## 9. 页面规格

### 9.1 Today

#### 页面目标
- 用户每天最高频打开的入口。
- 让用户快速理解“今天最重要的事是什么”。

#### 必备模块
- 顶部日期与概览
- 周视图日期切换
- 今日任务流 / 时间线
- 任务完成、重新打开、进入详情
- 快速筛选与排序入口
- 主按钮创建入口

#### 关键交互
- 点按日期切换 Today 内容。
- 点按任务进入详情或展开态。
- 快速完成任务。
- 切换筛选：全部 / 高优先级 / 进行中 / 已完成。

#### 结构与状态流
1. 顶部日期区
2. 周视图
3. 今日任务流 / 时间线
4. 底部或悬浮创建入口

- 打开应用 -> 进入 Today
- 点击日期 -> 切换焦点日
- 点击任务 -> 展开详情
- 完成任务 -> 卡片沉降到已完成状态

### 9.2 清单

#### 页面目标
- 承接任务收集、分类与整理。

#### 必备模块
- 系统清单：收集箱、今天、稍后
- 自定义清单列表
- 清单内任务列表
- 批量整理入口

#### 结构要点
- 页标题和筛选
- 系统清单 + 自定义清单主体
- 点击清单进入对应任务列表
- 批量整理通过底部工具条或顶部编辑态进入

### 9.3 项目

#### 页面目标
- 承接中周期任务聚合与推进。

#### 必备模块
- 项目卡片列表
- 项目状态：进行中 / 暂停 / 已完成
- 项目详情页
- 项目内任务聚合

#### 结构要点
- 顶部项目概览和状态筛选
- 主体为项目卡片列表
- 项目详情展示说明、关键任务、截止日期

### 9.4 日历

#### 页面目标
- 从时间维度查看任务分布与截止压力。

#### 必备模块
- 周视图
- 月视图
- 焦点日期任务列表
- 快速跳转今天

#### 结构要点
- 顶部为周 / 月切换与返回今天
- 中部为日历主体
- 下方为焦点日期任务列表
- 焦点日期贯穿周 / 月 / 详情上下文

### 9.5 我

#### 页面目标
- 承接个人设置与未来模式入口。

#### 必备模块
- 个人信息
- 通知设置
- 主题 / 深色模式 / 动效偏好
- 默认首页设置
- 未来双人模式入口占位

#### 结构要点
- 顶部为个人资料
- 中部为通知、主题、动效、默认视图设置
- 底部为未来双人模式入口或说明位

## 10. 必要二级页面
- 任务详情页
- 任务创建 / 编辑页
- 项目详情页
- 筛选 / 排序 / 分组面板
- 全局快捷创建入口

## 11. 当前实现结论
- 当前首页静态 UI 直接沿用为 Today 首页基础，不做推翻式重构。
- 当前仓库里的 `Decisions / Anniversaries` 导航壳子属于旧逻辑遗留，不应继续代表首版信息架构。
- 当前单人模式下不展示复杂协作信息。

## 12. 版本路线
- V1：单人模式 Todo 效率工具。
- V2：双人模式，通过邀请 / 分享形成 2 人协作空间，仍以 Todo 为核心。
- V3：多人 Space 作为远期方向，仅在底层架构中预留。

## 13. 未来扩展边界
- V2 可在 Today 顶部或我页加入轻量的单人 / 双人空间切换。
- V2 可支持共享清单、共享项目、共享任务可见性层。
- 当前不提前暴露 Space 选择页，不提前暴露多人 IA。

## 14. 成功标准
- 用户能在 10 秒内进入 Today 并理解当天重点。
- 快速新建与完成任务的路径足够短，可高频使用。
- 周 / 月视图切换自然，不影响任务上下文。
- 首页、详情、筛选切换保持明显的原生顺滑感。

## 15. Open Questions
- V1 的全局创建入口是否同时承载“新建项目”，还是先聚焦“新建任务”。
- “清单”是否长期保留一级导航，还是后续并入 Today 的整理视图。
- 快速创建入口最终是固定单按钮还是展开菜单，需要结合实现成本与手感验证。
- 决策与纪念日最终作为二级入口还是功能开关，需要在 V1 稳定后再定。

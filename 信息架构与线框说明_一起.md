# 信息架构与线框说明 · 一起 / Together

## 1. 结论
- 当前信息架构以单人 Todo 为主，不做双人优先展开。
- 当前首页静态 UI 保留，视为 Today 首页。
- 未来通过 Space 演进支持双人和多人，但当前产品层不强调 Space 概念。

## 2. 当前 MVP 信息架构

### 一级结构
- Today
- 清单
- 项目
- 日历
- 我

### 全局入口
- 快速创建任务
- 搜索
- 筛选 / 排序 / 分组
- 任务详情

## 3. 关键对象关系
- Today：今日焦点与时间线入口
- 清单：任务收集与分类整理
- 项目：中周期推进容器
- 日历：时间维度映射
- 我：个人偏好与未来模式入口

任务是整个 IA 的核心对象；清单、项目、日历只是不同观察角度。

## 4. Today 线框说明

### 当前结论
- 直接沿用现有首页 UI 骨架。

### 结构
1. 顶部日期区
2. 周视图
3. 今日任务流 / 时间线
4. 底部或悬浮创建入口

### 状态流
- 打开应用 -> 进入 Today
- 点击日期 -> 切换焦点日
- 点击任务 -> 展开详情
- 完成任务 -> 卡片沉降到已完成状态

### 动效重点
- 日期切换
- 卡片完成
- 卡片展开 / 收起
- 列表重排

## 5. 清单页线框说明
- 左上为页标题和筛选
- 主体为系统清单 + 自定义清单
- 点击清单进入该清单任务列表
- 批量整理通过底部工具条或顶部编辑态进入

## 6. 项目页线框说明
- 顶部为项目概览和状态筛选
- 主体为项目卡片列表
- 进入项目详情后展示项目说明、关键任务、截止日期

## 7. 日历页线框说明
- 顶部为周 / 月切换与返回今天
- 中部为日历主体
- 下方为焦点日期任务列表
- 保持焦点日期贯穿周 / 月 / 详情三种上下文

## 8. 我页线框说明
- 顶部为个人资料
- 中部为通知、主题、动效、默认视图设置
- 底部为未来双人模式入口或说明位

## 9. 未来演进

### V2
- 在不重做全局导航的前提下，引入双人模式
- 入口位置建议：
  - 我页
  - 或 Today 顶部轻量范围切换器

### V3
- 若有明确价值，再引入多人 Space
- 当前不提前暴露多人 IA

## 10. 不该出现的复杂化
- 不要在 V1 里提前做 Space 选择页
- 不要把 Today 做成信息过载的超级仪表盘
- 不要把双人模式的邀请、共享权限、纪念日、决策流塞进首页主路径

## 11. Open Questions
- V1 的“清单”是否单独一级导航，还是后续并入 Today 的整理视图，需要在代码层排期前再最终确认。
- 快速创建入口最终是固定单按钮还是展开菜单，需要结合实现成本与手感验证。

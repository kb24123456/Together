# Together UI/UX Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 audit (`docs/superpowers/specs/2026-04-19-ui-polish-audit.md`) 中 P0/P1/P2 共 9 类 ~100+ 细项逐批落地：先清 token + 可达性硬伤（Batch A），再补 haptic / 空态 / Dynamic Type（Batch B），最后 iPad / VoiceOver / 对比度收尾（Batch C）。

**Architecture:**
- Token 扩展走 `Together/Core/DesignSystem/AppTheme.swift` 单文件增量；所有 view 迁移走机械替换 + 视觉回归 build。
- 新增 pair-mode 视觉层借现有 `sessionStore.activeMode == .pair` 判定，新增 `AppTheme.colors.pairAccent` 系列 token + 首页顶部 badge。
- Haptic 扩展沿用 `HomeInteractionFeedback` 枚举，新增 `.delete()` / `.error()` / `.warning()` 静态方法，在真正触发副作用的 VM 边界调用。
- Empty state 不新写组件，扩 `EmptyStateCard` 支持可选 SF Symbol illustration slot，原有调用默认行为不变。
- Dynamic Type 迁移严格走 `AppTheme.typography.textStyle(_:weight:)` / `.sized(_:weight:)`，禁 `.font(.system(size:))` / `.font(.body)`。

**Tech Stack:** SwiftUI · SwiftData (17 models) · Swift Testing (`import Testing` + `@Test` + `#expect`) · Xcode 16 synchronized file groups · iOS 17+（少量 iOS 26 `glassEffect`）

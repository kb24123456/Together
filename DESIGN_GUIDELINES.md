# Together Design Guidelines

## 1. 文档职责
- 本文件是项目「一起 / Together」的唯一设计规范主文档。
- 负责：视觉基线、安全区域、页面设计规则、动效规则、Pencil 出图约束。
- 不负责：产品范围、信息架构和页面模块清单；这些以 `PRODUCT_SPEC.md` 为准。

## 2. 结论
- 本仓库所有 UI 设计和实现，默认按 iOS 原生设计规范执行。
- 当前首页 UI 保留，并作为单人模式 Today 首页的设计母版继续深化。
- 动效与交互丝滑度是核心体验组成，不是最后补的装饰层。

## 3. 适用范围
- Pencil 线框图和高保真视觉稿
- SwiftUI 页面实现
- UIKit 容器或桥接页面
- 动效设计、导航转场、悬浮操作按钮

## 4. 全局视觉基线

### 事实
- 当前项目默认视觉基线继续使用 `mobile-02-cleanminimal_light`。
- 现有首页 UI 的干净留白、轻层次、弱阴影、低饱和配色是正确方向。

### 设计结论
- 产品表达应是“效率优先、带一点温度”，不是“情侣感优先”。
- 允许保留黑 / 白 / 粉的品牌倾向，但只能作为强调色，不应破坏效率工具的清晰层级。
- 不允许转向厚重拟物、强发光、过度装饰或廉价甜腻风格。

## 5. 全局硬规则
1. 顶部内容不得侵入安全区域。
2. 底部内容不得侵入 Home Indicator 与系统手势区域。
3. 滚动内容与固定底栏必须分层。
4. 安全区域按运行时约束处理，不写死机型像素。
5. 自定义玻璃和半透明效果必须服从系统层级与可读性。
6. 英文与数字默认使用 Rounded 风格。
7. 全局字体系必须统一，不允许页面临时拼接字体方案。

## 6. SwiftUI 设计落地规则
1. iOS 17+ 优先使用 `safeAreaPadding`、`safeAreaInset`、`GeometryProxy.safeAreaInsets`。
2. 背景层可全屏延展，主要信息容器必须尊重安全区域。
3. FAB、顶部操作按钮、分段切换、悬浮胶囊按钮都必须保留足够触控热区。
4. 列表卡片和底部工具条的视觉层次要清楚，但不要靠重阴影和大模糊堆出“假高级”。
5. 原生 iOS 组件优先使用 iOS 26 液态玻璃原生能力；需要玻璃质感时，优先 `glassEffect`、`GlassEffectContainer` 和系统 glass button style，不手搓高饱和假玻璃。

## 7. Today 首页专项规则
- 当前首页 UI 作为 Today 首页基础，不推翻重画。
- 顶部日期、周视图、主内容区的层级要保持清晰。
- 单人模式下默认展示个人工作焦点；未来双人模式如需出现协作者信息，应以轻量头像或状态芯片附着，不应破坏当前骨架。
- 列表 / 时间线卡片应优先表达：
  - 时间
  - 标题
  - 次级信息（项目、位置、标签、提醒）
  - 状态
- 首页卡片高度由内容决定，不写死固定高度。
- 已完成任务的视觉降噪要明显，但仍要保留可回看性。

## 8. 日历与详情页规则
- 周视图与月视图的切换必须保持同一焦点日期。
- 详情页优先采用系统感 sheet 或卡片扩展过渡。
- 筛选、排序、分组面板优先使用轻量浮层，不要做复杂弹窗堆叠。

## 9. 动效规则

### 原则
- 原生
- 克制
- 连续
- 可读
- 稳定 60fps

### 优先场景
- 任务完成与状态切换
- 首页列表层级变化
- 周视图 / 月视图切换
- 主按钮展开 / 收起
- 任务详情展开 / 收起
- 筛选 / 排序 / 分组切换
- 未来单人 / 双人模式切换

### 禁止事项
- 禁止廉价、花哨、堆砌式动画。
- 禁止在动画过程中做重排序、重计算、同步 IO。
- 禁止为了转场效果让文字失焦、可读性下降或点击区域错位。

## 10. Pencil 出图规则
- 绘制 iPhone 页面时必须显式预留顶部和底部安全区域。
- Today 页面要先表达层级和节奏，再表达装饰。
- 若标注动效，必须写清：
  - 触发条件
  - 起止状态
  - 节奏类型
  - 是否有降级方案

## 11. Open Questions
- 是否需要为动效建立单独的 design token，例如 `duration / spring / elevation transition`。
- V2 双人模式的协作者信息在 Today 首页中以头像、标签还是轻量状态条出现。

# 一起 状态机说明

## 1. 事项状态机

### 1.1 状态
- `待确认`
- `进行中`
- `已完成`
- `未同意 / 无法完成`

### 1.2 文字流转
1. 发起事项后进入 `待确认`
2. 若对方反馈：
   - `愿意处理` -> `进行中`
   - `愿意一起处理` -> `进行中`
   - `知道了` -> `进行中`
   - `暂不方便` -> `未同意 / 无法完成`
   - `这次不合适` -> `未同意 / 无法完成`
3. `进行中` 状态下被标记完成 -> `已完成`

### 1.3 按执行归属拆分
- `由对方完成`
  - 对方愿意处理 -> 进行中
  - 对方暂不方便 / 这次不合适 -> 未同意 / 无法完成
- `双方共同完成`
  - 对方愿意一起处理 -> 进行中
  - 对方暂不方便 / 这次不合适 -> 未同意 / 无法完成
- `由我完成，对方知情`
  - 对方知道了 -> 进行中
  - 不走“是否愿意执行”的分支

### 1.4 异常与边界
- 未绑定时：
  - 允许创建本地草稿
  - 不允许真实发送
- 待确认无人反馈：
  - 保持待确认
- 已完成事项：
  - 首版不支持重新打开
- 未同意 / 无法完成：
  - 首版仅保留结果，不自动重试

## 2. 决策状态机

### 2.1 状态
- `待表态`
- `已达成一致`
- `暂未达成一致`
- `已归档`

### 2.2 文字流转
1. 创建决策 -> `待表态`
2. 双方表态完成后：
   - 若双方都 `赞同` -> `已达成一致`
   - 若任一方 `中立 / 否认` -> `暂未达成一致`
3. `暂未达成一致` 可：
   - 再次提醒，状态不变
   - 主动结束 -> `已归档`
4. `已达成一致` 可一键转为事项
   - 当前决策仍保留一致结果
   - 新建一个 Item 进入事项流

### 2.3 异常与边界
- 只有一方表态：
  - 仍属于 `待表态`
- 已归档：
  - 首版不支持恢复
- 转事项：
  - 首版只记录 `convertedItemID`
  - 不做双向复杂回写

## 3. 单人试用 -> 邀请绑定 -> 双人协作

### 3.1 状态
- `singleTrial`
- `invitePending`
- `inviteReceived`
- `paired`
- `unbound`

### 3.2 流转
1. 首次登录 -> `singleTrial`
2. 创建邀请 -> `invitePending`
3. 对方收到邀请 -> `inviteReceived`
4. 邀请被接受 -> `paired`
5. 双方任一侧触发解绑 -> `unbound`
6. 若重新绑定新对象：
   - 不是从 `unbound` 恢复旧数据
   - 而是创建新的 `PairSpace`

### 3.3 关键边界
- 单人试用期间：
  - 可以看示例、体验结构、创建轻草稿
  - 不能形成真实共享对象
- 一对一限制：
  - 一个活跃用户不能同时存在多个活跃 PairSpace
- 解绑后：
  - 旧数据不可迁移到新关系
  - 新关系数据从零开始

## 5. Assumptions / Open Questions

### 5.1 Assumptions
- 首版不引入“已归档事项”独立状态
- 首版不引入“决策撤回”和“事项重新打开”

### 5.2 Open Questions
- 若邀请超时或被拒绝，是否需要单独页面和重发机制，文档未明确
- 事项完成是否要求双方确认，文档未明确，当前按单次完成处理

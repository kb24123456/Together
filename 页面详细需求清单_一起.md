# 页面详细需求清单 · 一起 / Together

## 1. 当前 MVP 页面结论
- 一级结构建议：`Today / 清单 / 项目 / 日历 / 我`
- 当前首页静态 UI 直接沿用为 Today 首页基础，不做推翻式重构。
- 决策、纪念日不再作为当前一级导航主页面。

## 2. 全局要求
- 所有页面支持深色模式、SwiftUI Preview、mock data。
- 所有页面遵守顶部和底部安全区域。
- 所有关键状态切换都要设计原生、克制、稳定的动效。
- 所有列表页都要有空态、加载态、错误态。

## 3. Today 首页

### 页面目标
- 用户每天最高频打开的入口。
- 让用户快速理解“今天最重要的事是什么”。

### 必备模块
- 顶部日期与概览
- 周视图日期切换
- 今日任务流 / 时间线
- 任务完成、重新打开、进入详情
- 快速筛选与排序入口
- 主按钮创建入口

### 关键交互
- 点按日期切换 Today 内容
- 点按任务进入详情或展开态
- 快速完成任务
- 切换筛选：全部 / 高优先级 / 进行中 / 已完成

### 动效要求
- 日期切换：轻量层级切换，不做重翻页
- 任务完成：自然收束 + 轻触感反馈
- 任务详情展开：连续过渡，不割裂上下文
- 列表重排：有秩序感，不瞬移

### 未来扩展
- V2 可在顶部加入轻量的单人 / 双人工作空间切换
- 但当前单人模式下不展示复杂协作信息

## 4. 清单页

### 页面目标
- 承接任务收集、分类与整理

### 必备模块
- 系统清单：收集箱、今天、稍后
- 自定义清单列表
- 清单内任务列表
- 批量整理入口

### 动效要求
- 清单切换时保留选中态连贯感
- 批量整理与拖拽排序保持稳定反馈

### 未来扩展
- V2 可支持共享清单，但当前不实现共享逻辑

## 5. 项目页

### 页面目标
- 承接中周期任务聚合与推进

### 必备模块
- 项目卡片列表
- 项目状态：进行中 / 暂停 / 已完成
- 项目详情页
- 项目内任务聚合

### 动效要求
- 项目卡片展开 / 收起应有连续层级过渡
- 项目状态切换要有明确的视觉沉降或提升

### 未来扩展
- V2 可支持双人共享项目

## 6. 日历页

### 页面目标
- 从时间维度查看任务分布与截止压力

### 必备模块
- 周视图
- 月视图
- 焦点日期任务列表
- 快速跳转今天

### 动效要求
- 周 / 月切换必须保留焦点日期
- 日期选中态要有明显但克制的过渡

### 未来扩展
- V2 可叠加共享任务的可见性层，但当前不做

## 7. 我

### 页面目标
- 承接个人设置与未来模式入口

### 必备模块
- 个人信息
- 通知设置
- 主题 / 深色模式 / 动效偏好
- 默认首页设置
- 未来双人模式入口占位

### 动效要求
- 设置项切换与开关反馈保持系统感
- 未来模式入口如果出现，不要打断当前单人主链路

## 8. 必要二级页面
- 任务详情页
- 任务创建 / 编辑页
- 项目详情页
- 筛选 / 排序 / 分组面板
- 全局快捷创建入口

## 9. 非当前 MVP 主页面
- 决策页：降级为后续附加模块
- 纪念日页：降级为后续附加模块
- 邀请 / 绑定 / 关系页：V2 以后再进入主开发节奏

## 10. 当前实现备注
- 当前仓库已有首页静态 UI，可直接作为 Today 首页继续开发。
- 当前仓库里的 `Decisions / Anniversaries` 导航壳子属于旧逻辑遗留，不应继续代表首版信息架构。
